# Memory Sweep Candidates

Auto-generated files from the enterprise memory sweep workflow.

Each file named YYYY-MM-DD.md contains raw learning signals extracted from
asecoat-enabled repos, formatted for human review.

## Workflow

1. Weekly sweep opens a PR with a new YYYY-MM-DD.md file
2. Reviewer (memory-curator agent or memory steward) evaluates each candidate
3. Worthy candidates are promoted to {org}/basecoat-memory via separate PR
4. Reviewed files are archived here for audit history

See docs/memory/enlistment.md for how repos opt in.
