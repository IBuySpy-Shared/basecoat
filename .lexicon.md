## 1. Vocabulary Registry

| Canonical term | Definition | Do not use | Notes |
|---|---|---|---|
| **BaseCoat** | The product name in prose, headings, and documentation | Base Coat (space), base-coat, BASECOAT | One word, camelcase |
| **basecoat** | The repository identifier in code, URLs, and file paths | Base Coat, BaseCoat, base-coat | All lowercase, no separator |
| **base-coat** | The consumer install directory path only: `.github/base-coat/` | All other contexts | Hyphenated form is path-only |
| **agent** | A `.agent.md` file that defines an autonomous Copilot role with inputs, workflow, and outputs | bot, assistant, module, plugin | Singular: agent; plural: agents |
| **skill** | A `SKILL.md` file in `skills/<name>/` providing reference knowledge, templates, or tools | plugin, extension, library, module | Singular: skill; plural: skills |
| **instruction** | An `.instructions.md` file loaded as ambient context by Copilot | rule, guideline, policy, prompt | Singular: instruction; plural: instructions |
| **prompt** | A `.prompt.md` file providing a structured invocation template | template (ambiguous), script | Singular: prompt; plural: prompts |
| **asset** | Any agent, skill, instruction, or prompt | resource, artifact, component, file | Collective noun for all four primitives |
| **consumer** | A repository that runs `sync.ps1` / `sync.sh` to adopt BaseCoat assets | user, client, adopter, tenant | The team or repo that uses BaseCoat |
| **contributor** | Someone who adds or improves assets in the BaseCoat repository | committer, author, developer | Internal or external; opens PRs |
| **steward** | The team that reviews memory candidates and governs the shared memory repo | admin, moderator, owner, gatekeeper | Specifically for memory governance |
| **sync** | Running `sync.ps1` / `sync.sh` to overlay BaseCoat assets into a consumer repo | install, deploy, import, apply | The act of adopting or refreshing assets |
| **overlay** | The result of sync: BaseCoat files placed into `.github/` of a consumer repo | installation, deployment | The files placed, not the process |
| **drift** | When a consumer repo's BaseCoat version falls behind the upstream release | lag, delay, stale | Version drift, not content drift |
| **hot index** | The L2 memory layer: a curated fast-load summary of key session facts | cache, shortcut, index | Loaded at session start; ≤1,500 tokens |
| **memory candidate** | A learning identified for potential promotion to shared memory | pattern, fact, observation | Must pass 4-point scope check before promotion |
| **promote** | Moving a memory candidate from session/local storage to shared memory | publish, commit, merge, submit | Steward action after review |
| **wave** | A batch of agents dispatched concurrently in a fleet sprint | group, batch, tier | Max 3-4 agents per wave |

## 3. Ontology

### Is-a relationships

```
Asset
  ├─ Agent: an autonomous role definition; reads inputs, runs workflow, produces output
  ├─ Skill: a reference document; provides knowledge, templates, or tools to agents
  ├─ Instruction: an ambient rule file; loaded by Copilot for matching file types
  └─ Prompt: a structured invocation template; triggers a specific workflow
```

### Depends-on relationships

| Concept | Depends on | Nature |
|---|---|---|
| Agent | Skill (optional) | An agent may reference skills via `allowed_skills`; skill provides the knowledge |
| Agent | Instruction | Instructions are loaded ambient; agents benefit from them without explicit reference |
| Consumer repo | Sync | A consumer only receives assets after `sync.ps1` / `sync.sh` runs |
| Memory candidate | Session | Candidates come from session insights; they don't exist independently |
| Hot index | Memory candidate (promoted) | The hot index is populated from promoted memories via weekly sync |
| Wave | Agent (multiple) | A wave is a concurrent dispatch of N agents; requires N agent definitions |

### Mutually exclusive concepts

- **Agent** vs **Skill**: an agent drives workflow and produces output; a skill provides reference knowledge consumed by agents. A skill does not execute; an agent does.
- **Instruction** vs **Prompt**: instructions are always-on ambient rules; prompts are explicit invocation templates used on demand.
- **Consumer** vs **Contributor**: a consumer runs sync; a contributor opens PRs. One org can be both simultaneously.
- **Sync** vs **Promote**: sync overlays BaseCoat assets into a consumer repo; promote moves a memory candidate into shared memory. Different directions, different actors.

### Composition relationships

| Whole | Made of | Constraint |
|---|---|---|
| BaseCoat library | Agents + Skills + Instructions + Prompts | All four required; none optional |
| Consumer overlay | `.github/agents/` + `.github/skills/` + `.github/instructions/` + `.github/prompts/` | Created by sync; not committed by hand |
| Wave | 3–4 concurrent agents | Hard upper limit: 5+ causes 429 rate limit |
| Sprint | N waves + retrospective | No fixed wave count; waves determined by dependency map |

## 4. Theme and Vibe

### Personality

> **Precise · Pragmatic · Enterprise-grade**

### Tone of voice

| Dimension | Direction | Example |
|---|---|---|
| Formality | Semi-formal | "Run `sync.ps1` to overlay assets." not "Just run sync!" |
| Density | Balanced | Lead with the point; follow with context |
| Technical register | Practitioner | Assumes Git and Copilot familiarity; explains BaseCoat-specific concepts |
| Perspective | Second person | "You can sync any repo in under 5 minutes." |
| Hedging | Minimal | State what the tool does; don't over-qualify or apologize |

### Anchoring metaphors

These metaphors are coherent with the brand and acceptable to use:

- **Base coat as foundation**: a layer applied before the finish coat — ambient, invisible, supportive. Works for describing instructions and the overlay pattern.
- **Adherence**: instructions adhere uniformly; per-agent duplication doesn't stick. Works for explaining why centralized instructions beat copy-paste.
- **Tacky window**: new repos and onboardings are the moment when guidance lands before habits form. Works for explaining the value of early adoption.
- **Bare patches / drift**: what happens without a base coat — gaps between teams, inconsistent behavior, rework. Works for explaining version drift and coverage gaps.

### Anti-references

- ❌ **Paint/chemistry product vibe**: do not reference primer, coats, solvents, drying time, mixing ratios — the metaphor should be subtle, not a theme park.
- ❌ **Startup hype**: avoid "revolutionary", "game-changing", "10x", "supercharge". BaseCoat is infrastructure; it should sound like it.
- ❌ **Academic formality**: avoid passive voice throughout, nominalization ("the utilization of"), and citation-style prose. This is a practitioner tool, not a paper.
- ❌ **Overcasual**: avoid "just", "simply", "easy", "obvious". These words minimize the reader's legitimate complexity.

### Anti-vocabulary

| Avoid | Because | Use instead |
|---|---|---|
| "just run sync" | Dismissive; sync has real prerequisites | "Run `sync.ps1`" |
| "plugin" | Implies an extension model; BaseCoat uses overlay | "skill" or "asset" |
| "template" (unqualified) | Ambiguous — could mean agent, prompt, or file template | "prompt template", "agent template", "file template" |
| "install" (for sync) | Implies a package manager; BaseCoat is file-based | "sync" or "overlay" |
| "Copilot instructions" (generic) | Confuses Copilot feature with BaseCoat instructions | "BaseCoat instructions" or "instruction files" |
