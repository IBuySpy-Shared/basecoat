# BaseCoat — Design Context

## Design Context

### Users

Platform engineers, senior developers, and GitHub org administrators at enterprise organizations. They arrive via a GitHub search, a team Slack recommendation, or a manager directive. Context: evaluating whether BaseCoat is worth adopting for their team, or looking for a specific agent/skill to copy. They are technical, skeptical, and time-poor — they need to understand value in 30 seconds and find the right asset in under 2 minutes.

### Brand Personality

**Authoritative · Precise · Enabling**

BaseCoat is a foundation layer — it does the unglamorous work that makes everything else possible. Like a good base coat in painting: invisible when working correctly, critical when it's not there. Voice: direct, confident, no marketing fluff. It speaks to engineers, not executives.

### Aesthetic Direction

- **Visual tone**: Professional and technical. Clean information density. GitHub-adjacent without being a GitHub clone.
- **Color**: Deep purple primary (`deep-purple`) — distinctive from GitHub blue, serious but not cold. Conveys "enterprise AI framework" rather than "startup tool."
- **References**: GitHub docs (information density), Stripe docs (clarity), Linear (precision). NOT: marketing sites with full-bleed hero images, cartoon illustrations, or heavy gradients.
- **Theme**: Light mode default with dark mode available. Code-heavy content demands good contrast.
- **Logo**: `material/layers` icon — the stacked layers metaphor fits BaseCoat's role as a foundation layer beneath Copilot customizations.

### Design Principles

1. **Content before chrome** — nav and layout serve the content, not the other way around. Every pixel earns its place.
2. **First screen sells** — the landing page must answer "what is this and why do I care?" before the fold.
3. **Progressive disclosure** — home page → section overview → detail. Never dump everything at once.
4. **Link to source, don't duplicate** — docs files that mirror repo files should link to GitHub rather than maintain parallel copies.
5. **Consistent wayfinding** — no dead-end pages, no nav that navigates away without warning, no duplicate entries.
