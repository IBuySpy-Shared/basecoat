# Agent Registry Design

**Issue:** #478
**Status:** Approved
**Sprint:** 8, Wave 1

## Overview

The agent registry is an in-memory catalog of all agents available in Basecoat. It enables the `/basecoat` plugin to discover, validate, and route commands to the correct agent without enumerating the file system on every invocation.

## Registry Format

Agents are described in a single JSON file (`basecoat-registry.json`) that is generated from the agent markdown frontmatter at build time, bundled into the plugin, and updated via a background refresh mechanism.

### Top-Level Schema

```json
{
  "version": "1.0.0",
  "generated": "2026-05-07T00:00:00Z",
  "agents": {
    "<agent-id>": { ... }
  }
}
```

### Agent Entry Schema

```json
{
  "id": "solution-architect",
  "name": "Solution Architect",
  "description": "System design, C4 diagrams, ADRs, technology selection",
  "file": "agents/solution-architect.agent.md",
  "keywords": ["architecture", "design", "c4", "adr", "system"],
  "capabilities": ["backend", "frontend", "infrastructure"],
  "model": "claude-sonnet-4.6",
  "maturity": "production",
  "category": "Architecture & Design"
}
```

### Full JSON Schema (JSON Schema Draft 7)

See `schema/agent-registry.schema.json`.

## Caching Strategy

| Concern | Decision |
|---|---|
| Storage | In-memory `Map<string, AgentEntry>` |
| TTL | Configurable (default: 3600 seconds = 1 hour) |
| Invalidation | TTL expiry + manual `registry.refresh()` |
| Seed data | Bundled `basecoat-registry.json` loaded at startup |
| Network update | Optional: fetch updated registry from GitHub raw URL |
| Fallback | On network error, continue with bundled/cached data |

### Cache Lifecycle

```
startup
  └─► load bundled registry (sync, <5ms)
  └─► start background TTL timer
  └─► [optional] fetch remote update (async, non-blocking)

cache hit (age < TTL)
  └─► return cached entry (<1ms)

cache miss / TTL expired
  └─► attempt remote fetch
      ├─► success: refresh cache, reset TTL
      └─► failure: log warning, serve stale cache
```

## Agent Discovery Algorithm

### Exact Match (primary)

Given input `/basecoat solution-architect design the API`:
1. Normalize agent-id: `solution-architect`
2. Look up in registry map: O(1)
3. Return entry or `null`

### Fuzzy Match (fallback)

If exact match fails:
1. Tokenize input agent-id (split on `-`, `_`, spaces)
2. Score each registry entry: `score = matchedTokens / totalTokens`
3. Return entry with highest score ≥ 0.5, or `null` if no match

### Disambiguation

If multiple entries score equally:
1. Return top-3 candidates as suggestions
2. Prompt user to clarify (output: `Did you mean: solution-architect, security-analyst, sprint-planner?`)

## Performance Requirements

| Operation | Target | Measured Against |
|---|---|---|
| Registry load (bundled) | < 10ms | startup |
| Cache hit lookup | < 1ms | per invocation |
| Fuzzy search (73 agents) | < 20ms | per invocation |
| Remote registry fetch | < 2000ms | background |

## Error Handling

| Scenario | Behavior |
|---|---|
| Registry file missing | Throw `RegistryLoadError` at startup |
| Network fetch timeout | Log warn, serve stale cache |
| Agent not found | Return `null`, caller handles user message |
| Malformed registry JSON | Log error, serve previous valid cache |

## Network Error Handling

The registry loader follows this retry strategy for remote fetches:

- Attempt 1: immediate
- Attempt 2: after 1s
- Attempt 3: after 4s
- After 3 failures: mark remote as unavailable for 1 TTL period

## Registry Update Mechanism

The bundled `basecoat-registry.json` is generated by `scripts/generate-registry.ps1` as part of the build pipeline. Remote updates can be fetched from:

```
https://raw.githubusercontent.com/IBuySpy-Shared/basecoat/main/plugins/copilot-cli-plugin/schema/basecoat-registry.json
```

This allows the plugin to pick up new agents without a reinstall, while always having a valid offline fallback.

## TypeScript Interfaces

```typescript
export interface AgentEntry {
  id: string;
  name: string;
  description: string;
  file: string;
  keywords: string[];
  capabilities: string[];
  model: string;
  maturity: 'production' | 'beta' | 'experimental';
  category: string;
}

export interface AgentRegistry {
  version: string;
  generated: string;
  agents: Record<string, AgentEntry>;
}

export interface RegistryCache {
  data: AgentRegistry;
  loadedAt: number;
  ttlMs: number;
  isStale(): boolean;
}

export interface RegistryLookupResult {
  agent: AgentEntry | null;
  confidence: number;
  suggestions: AgentEntry[];
}
```
